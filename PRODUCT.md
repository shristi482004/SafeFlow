# Product

## Register

product

## Users

**Stadium operators (Staff, Admin)**: Security coordinators and event safety officers managing crowd flow, incident response, and resource dispatch during live cricket events. High-stakes context — scanning fast, making calls under pressure, limited tolerance for ambiguity.

**Fans**: Spectators at cricket stadiums in India. May need to trigger an SOS, find a safe exit, or locate a lost family member. Non-technical, potentially stressed, in a noisy physical environment. Multilingual: English, Hindi, Tamil, Telugu.

## Product Purpose

SafeFlow is a real-time AI stadium safety command center. It observes crowd conditions, reasons about emergencies with a Gemini 3-agent pipeline, coordinates resource dispatch (stewards, medical, police), and guides fans through safe exits and SOS flows. The system exists to reduce harm at large-scale events — its success is measured in response time, clarity of action, and calm under pressure.

## Brand Personality

Trusted public infrastructure with calm authority.

Communicates clearly, reduces stress, guides users through situations confidently. Never robotic, never alarmist, never overly technical, never overly friendly. Feels like a well-maintained civic system — present when needed, invisible when not.

## Anti-references

- Sci-fi control room aesthetics: no hollow-tech glows, no cinematic HUD grids, no animated ring loaders that exist to look impressive
- Generic SaaS dashboards: no card-icon-text grids, no startup gradients, no onboarding checklists dressed up as a safety tool
- Wellness-style language: no soft pastels, no rounded-bubbly type, no phrasing that sounds like a meditation app in an emergency

## Design Principles

1. **Clarity is the emergency feature.** When an incident occurs, every element on screen should reduce cognitive load. Nothing decorative earns space during active response.

2. **Calm signals competence.** Restraint communicates reliability. Visual noise — blinking, glow overload, unnecessary color — reads as instability, not urgency.

3. **The tool disappears; the action doesn't.** Interface chrome should recede. The operator's attention should be on what's happening in the stadium, not on the UI itself.

4. **Language is infrastructure, not a feature.** Multilingual support (English, Hindi, Tamil, Telugu) is baseline, not optional. Fan-facing copy at the same clarity standard as staff-facing copy.

5. **Trust is earned in quiet moments.** The idle state should feel as reliable and considered as the alert state. An interface that only looks serious during emergencies doesn't inspire confidence in the system.

## Accessibility & Inclusion

- Multilingual: en-IN, hi-IN, ta-IN, te-IN (voice + text)
- Large text mode (user-selectable)
- Simplified interface mode (reduced information density for fans)
- Dark and light themes (persistent, user preference)
- Target WCAG AA for all interactive elements and color contrast
